!!!Принцип работы бота!!!
Во время первого запуска бот запоминает ID пользователя, который его запустил.
Чтобы начать мониторинг документов необходимо активировать команду /RUN и начнется парсинг необходимых сайтов
раз в какой то промежуток времени, каждый раз после парсинга будет приходит сообщение всем у кого бот активирован,
имеется два типа сообщений, первый - если бот найдет соответствия с ключевыми словами, то отправит ссылку на документ,
второй - если соответствий не найдено, то отправит сообщение "Новых документов нет", так же всем у кого бот активирован.
Для того, чтобы отключить бота для пользователя, необходимо просто остановить бота в аккаунте пользователя, чтобы
полностью отключить мониторинг необходимо отключить бота программно.
Так же в боте имеется панель администратора, доступ к панели можно получить при помощи команды /admin.
Панель администратора дает возможность для трех действий:
	1 - Добавить новые ключевые слова. Чтобы это сделать необходимо нажать на кнопку /Добавить
	и ввести ключевые слова через запятую (Например: Генеральный план, ГП, Сервитут), также имеется
	возможность для отмены действия, просто введите "отмена" и можете выбирать другое действие;
	
	2 - Можно посмотреть на список ключевых слов, по которым в данный момент введется поиск, сделать
	это можно просто нажав на кнопку /Посмотреть и вам выведется список всех ключевых слов;

	3 - Удалить ключевые слова, Чтобы это сделать необходимо нажать на кнопку /Удалить и ввести через запятую
	ключевые слова, которые хотите удалить (Например: Генеральный план, ГП, Сервитут). Если вы ввели ключевое
	слово, которого нет в списке, то вам выведется сообщение "Данные слова не получилось удалить: удалить".
	Также имеется возможность для отмены действия, просто введите "отмена" и можете выбирать другое действие

!!!Как устроен бот!!!
Бот написан на Python 3.9, для корректной работы необходимо установить любую версию Python3.9 или Python 3.9+.
Чтобы убедиться что у вас установлена необходимая версия Python ведите в консоли для win 'python' для linux 'python3'
и увидите вашу версию, если версия не указана, то у вас не установлен python, если же она ниже 3.9, то советую
обновить до более высокой. Чтобы запустить бота под win, перейдите в папку с ботом, установите необходимые библиотеки
при помощи команды "pip install <название библиотеки>" и запустите команду "python BOT.py".
Чтобы запустить бота под linux, перейдите в папку с ботом, установите необходимые библиотеки при помощи команды
"pip3 install -U <название библиотеки>" и запустите команду "python3 BOT.py". Если у вас вышло надпись "Бот вышел в сеть",
значит бот работает.
 
Запуск бота производится из файла BOT.py

Чтобы изменить частоту проверок бота, откройте файл BOT.py текстовым редактором, желательно обычном блокнотом,
в строке 67 имеется запись "await asyncio.sleep(86400) # Время сна, если необходимо изменить частоту проверки, то необходимо изменить число, время в секундах"
необходимо установить необходимое число, вместо "(86400)", это число означает сколько секунд парсер будет спать перед  тем как
заново проверять записи.

Бот написан при помощи следующих библиотек:
aiogram
asyncio
requests
bs4
lxml

Бот состоит из 6-и модулей и 3 файлов:
craate_bot.py - Модуль инициализации бота;

BOT.py - основной модуль работы бота, через данный модуль происходит запуск бота. В нем прописаны стартовые команды,
	запуск парсера и объявление кнопок панели админа;

admin_kb.py - модуль настройки кнопок, каждая кнопка вызвана методом KeyboardButton и добавлена в клавиатуру при помощи
	ReplyKeyboardMarkup;

add_delete.py - в данном модуле прописаны все действия каждой из команд панели админа;

parser_website.py - модуль сбора всех документов, которые имеется на сайте;

settings.py - В данном модуле хранится токен бота;

keywords.txt - файл хранения ключевых слов, каждое новое ключевое слово хранится с новой строки;

filtr.txt - в данном файле хранятся названия всех документов, которые подходят под ключевые слова;

users.txt - файл хранения ID всех пользователей, которые запускали бота.
